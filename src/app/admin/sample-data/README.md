**Why some files have fewer than 50 records**: catalogs like Role Templates, Email Templates, and Notification Rules represent *configuration*, not customer-generated transactions — a real platform has a fixed, curated set of these (you don't need 50 different "password reset" email templates). Branding Overrides and Admin Team are naturally bounded by how many Enterprise orgs and internal staff exist. Every module that represents an ongoing stream of activity (audit logs, tickets, announcements, API keys, webhooks, backups, security events) has 50+ records as requested.

**To use in the React app**: copy the relevant JSON into the matching `src/features/<name>/<name>.mock.ts` file (or `import` the JSON directly and cast it to the TypeScript interface from `MODULE_SPEC.md` / `src/types/domain.ts`).
